# MyST 全语法详解（写作者专业指南）

本指南总结所有 MyST Markdown（最新版）推荐的写法，  
解释每种 Block 的：

- 用途
- 正确写法
- 常犯错误
- 注意事项

结合 MOF（Metal–Organic Framework）场景给出例子。

全部使用 **推荐的 `:::` fenced directives** 格式。

---

# 1. Admonitions（提示块）

适用于：  
- 解释 MOF 概念  
- 做出友情提示  
- 提醒注意事项  
- 给予背景知识  

可用类型：  
`note`, `tip`, `warning`, `important`, `hint`, `admonition`

---

提供简短提示、解释 MOF 背景知识。

这里是提示内容。
:::

:::{tip}
ZIF 系列 MOF 因其类似沸石拓扑结构而得名。

2. Figures（图片）

推荐写法：

:align: center
说明文字

3. Math（公式）

用途：
	•	吸附等温线方程
	•	晶胞体积计算
	•	孔隙率公式

行内公式：

孔隙率定义为 $\phi = V_\text{void}/V_\text{cell}$。

块级公式：

\phi = \frac{V_\text{void}}{V_\text{cell}}

复杂对齐：

\begin{aligned}
q(P) &= q_\max \frac{bP}{1 + bP} \\
K &= \frac{q}{P}
\end{aligned}

5. Code（代码块）

分两类：

5.1 静态代码（不执行）

:::{code} python
print(“这段代码不会被执行”)
:::

适用于：
解释概念 / 示例代码片段

⸻

5.2 可执行 code-cell（执行 Python）

:::{code-cell} python
print("此代码会被执行")
:::

常见错误：

```{code-cell}
print("❌ 老语法，不推荐")

---

## 5.3 添加元数据

例如隐藏输入：

:::{code-cell} python
:tags: hide-input
…
:::

常用 tags：

| tag | 含义 |
|-----|------|
| hide-input | 隐藏代码，只显示输出 |
| hide-output | 隐藏输出 |
| remove-input | 最终页面完全移除输入 |
| remove-output | 完全移除输出 |
| raises | 指定必须抛出的异常 |
| cache | 允许执行缓存 |

# 6. Cross-References（交叉引用）

用途：  
引用图 / 表 / 段落。

步骤：

1. 给 block 添加名字：

:::{figure} fig.png
:name: fig-mof
…
:::

2. 在正文引用：

如图 {ref}fig-mof 所示。

# 9. Citations（文献引用）

流程：

1. 在 `.bib` 文件定义文献  
2. 文章中引用：

如文献 {cite}yaghi2003reticular 所示

3. 独立引用块：

:::{cite} yaghi2003reticular
:::

# 10. Proofs / Theorems（定理与证明）

适用于材料理论部分。

### 示例：

:::{theorem} MOF 稳定性判据
若金属节点配位数足够大，则 MOF 一般更稳定。
:::

:::{proof}
高配位数限制了框架结构的变形自由度，从而提升稳定性。
:::
